## ✨ 核心功能

### 任務管理
- ✅ 創建、編輯、刪除、複製任務
- 🏷️ 標籤系統（分類和快速搜尋）
- 📅 日期設定（具體日期、未排程、不限期）
- ⚡ 快速日期選擇（今天、明天、後天、下週一～日）
- ⏱️ Block 時間估計（10 分鐘內、30 分鐘/1 block、多 blocks、非連續）
- 📊 任務統計（完成率、時間節省/延遲分析）
- 📝 下一步建議（完成任務時記錄反思與計劃）

### 分類系統
- 💼 **事業**（Career - 工作/學習）
- ⚡ **充電活動**（Recharge - 休閒/運動）
- 🏠 **生活必須**（Life - 日常事務）
- 🔍 **復盤**（Review - 回顧/反思）

### 視圖和過濾
- 📊 所有任務
- 📅 今天
- 📌 最近到期
- ✅ 已完成
- ⚠️ 逾期
- 按分類和標籤過濾

### 排序功能
- 拖曳排序（Drag & Drop）
- 固定順序
- 時間追蹤（實際完成時間 vs 預計時間）

### 迴圈群組
- 定時迴圈任務（自訂間隔天數 + 結束日期）
- 任務群組管理（群組內拖曳排序、自動日期計算）
- 完成後自動建立下次任務

### 完成時間分析
- **一般**：套用預計時間
- **⏩ 節省**：預計時間 - 實際時間（賺得時間）
- **⚠️ 延遲**：實際時間 - 預計時間（超支時間）
- **⏭️ 超前**：未到截止日期即完成

### 同步備份
- 🔗 **設定同步資料夾** - 連結本地資料夾（OneDrive/雲端）
- ⬆️ **匯出同步** - 寫入 JSON 至同步資料夾
- ⬇️ **匯入同步** - 從同步資料夾讀取
- ⬇️ **下載備份** - 手動匯出 JSON 檔案
- ⬆️ **導入備份** - 手動匯入 JSON 檔案

### 其他功能
- 📦 **壓縮任務** - 整理過期或已完成任務
- 🗑️ **清理已完成** - 批次刪除已完成任務
- 📊 **時間統計** - 匯總完成時間與效率分析

### UI/UX
- 🌙 深色主題（低眼疲勞配色）
- 📱 **響應式設計** - 完全適配手機（768px 斷點）
- ☰ **可收合側邊欄** - 手機優化
- 🎨 卡片式任務顯示，支援拖曳視覺回饋

## 文件結構

**TodoMaster.html** 包含三部分（共約 3810 行）：

1. **HTML** (前 ~1220 行)
   - 側邊欄菜單（視圖、分類、標籤、選項、同步備份）

3. **JavaScript** (~2590 行)
   - `TodoApp` 類：任務 CRUD、UI 渲染、事件處理
   - `localStorage` 持久化
   - 任務類型系統（積累型/目標型/超前型）
   - Block 分段計時（`blockTimeModal`）
   - 同步備份（File System Access API）
   - `downloadBackup()` / `uploadBackup()` 手動備份
   - 時間統計（每日趨勢圖、日曆視圖、分類分析）

## 維護原則

1. **單一文件** - 所有代碼在 `TodoMaster.html` 中
2. **零依賴** - 純 HTML5 + JavaScript，無外部庫
3. **本地存儲** - 所有數據在 `localStorage`，無伺服器
4. **隱私優先** - 用戶數據不上傳任何地方
5. **手機友好** - 響應式設計，768px 斷點

## 版本管理規範

### 版本號規則
- **Major.Minor** 格式（如 v3.0、v3.1、v4.0）
- **主版本**（X.0）：重大功能變更、UI 重設計、結構重構
- **副版本**（X.Y）：bug 修復、小功能新增、性能優化
- **修訂版本**（X.Y.Z）：緊急 bug 修復

### 備份檔案命名規範
```
標準格式：TodoMaster_{版本號}_{YYYY-MM-DD}.html

範例：
  TodoMaster_v3.4_2026-05-12.html  ← 目前版本
  TodoMaster_v3.5_2026-05-XX.html  ← 下一版本
```

## 限制與注意

- **localStorage 限制**：約 5-10MB，支援 1000+ 個任務
- **隱私模式**：隱身模式無法持久化數據
- **同步備份**：需瀏覽器支援 File System Access API（Chrome/Edge）
- **舊瀏覽器**：需支援 ES6 的現代瀏覽器

## ❓ 常見問題

**Q: 修改 HTML 後，需要上傳到伺服器嗎？**  
A: 不需要。使用者直接打開本地 TodoMaster.html 即可，所有數據存在瀏覽器本地。

**Q: 如何跨設備同步資料？**  
A: 使用「設定同步資料夾」功能，將同步資料夾指向 OneDrive 或其他雲端同步的本地目錄，再透過匯出/匯入同步。

**Q: 備份文件要保留多久？**  
A: 建議保留最近 10 個版本，定期清理過期備份。

**Q: 如何確保不會丟失使用者的任務數據？**  
A: 新版本需完全相容舊版本的 localStorage 格式，做好數據遷移測試。
